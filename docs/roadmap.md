# Roadmap

This project is still early. The roadmap is intentionally narrow: make the current Jira Server/Data Center migration flow safer, clearer, and easier to operate before expanding scope.

## Current priorities

- strengthen dry-run and apply-mode confidence
- improve operator docs and sample artifacts
- harden CI and test coverage around command parsing, config loading, and report generation
- tighten release hygiene for public users

## Next likely improvements

- more fixture-backed tests for migration edge cases
- better example artifacts for pre-migrate, migrate, and post-migrate phases
- clearer reporting around skipped items and manual prerequisites
- easier installation paths for public users, potentially including Homebrew

## Explicitly not planned right now

- Jira Cloud support
- broad Jira migration features outside Advanced Roadmaps teams and related references
- plan or program creation automation
- UI-driven workflows

## How to interpret this roadmap

Items here are directional, not guaranteed delivery commitments. Migration safety and reviewability take priority over scope growth.
